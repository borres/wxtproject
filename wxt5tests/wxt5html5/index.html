HTML5

The HTML5 tags may not work properly in all browsers

There are some issues with HTML5 in WXT.

WXT is built to rely mainly on wellformed XML-input. HTML5 does not have to be wellformed, but it can be.

WXT use tidy [1] as a helper if it runs across malformed (X)HTML. Tidy does not work properly with the new HTML5-tags.

HTML5 has some attribute that does not require values, example: 

  <p contenteditable> write here </p>

Any value assigned to contenteditable will turn it on, but it will not be validated as HTML5
